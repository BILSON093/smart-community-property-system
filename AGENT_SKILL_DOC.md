# AI Agent 智能助手 - 能力与技能文档

## 一、什么是 Agent？

Agent（智能体）是比普通AI客服更强大的AI助手。普通AI客服**只能回答问题**，而Agent**还能执行操作**。

| 对比项 | AI客服 `/ai/chat` | Agent `/agent/chat` |
|--------|-------------------|---------------------|
| 回答问题 | ✅ | ✅ |
| 查询数据 | ❌ 只能告诉你去哪查 | ✅ 直接帮你查 |
| 创建工单 | ❌ 只能说"请联系物业" | ✅ 直接帮你创建 |
| 缴纳费用 | ❌ 只能说"请前往缴费页面" | ✅ 直接帮你缴费 |
| 派单催缴 | ❌ | ✅ 管理员专属 |
| 工具调用 | 无 | 14个工具可调用 |

## 二、使用入口

### 管理员端（Web后台）
- 登录管理后台 → 侧边栏 → **AI智能助手**
- 路径：`/agent-chat`
- 拥有全部14个工具的权限

### 业主端（Web网页）
- 登录业主网页 → 顶部导航栏 → **智能助手**
- 路径：`/home/chat`
- 默认进入智能助手模式，顶部三个按钮可切换：**智能助手** / **AI客服** / **人工**
- 拥有10个业主工具的权限

### 业主端（微信小程序）
- 打开小程序 → 底部Tab **AI客服** → 点击按钮切换到 **智能助手** 模式
- 默认进入智能助手模式，点击按钮可切换：智能助手 → AI客服 → 人工客服
- 拥有10个业主工具的权限

### API接口
```
POST /api/agent/chat
Headers: Authorization: Bearer <token>
Body: {
  "message": "帮我看看系统概况",
  "history": []  // 可选，对话历史
}
```

## 三、技能清单（17个Skill）

### 🏠 业主工具（12个）

| # | 工具名 | 功能 | 示例触发语 | 说明 |
|---|--------|------|-----------|------|
| 1 | `query_unpaid_fees` | 查询欠费 | "我欠了多少钱？" | 列出所有未缴费项目，含fee_id供缴费使用 |
| 2 | `create_repair` | 创建报修 | "我家水管漏水了" | 自动创建报修工单，状态为"待审核" |
| 3 | `query_repair_status` | 查询报修进度 | "我的报修处理了吗？" | 显示最近5条报修记录及状态 |
| 4 | `pay_fee` | 缴纳费用 | "帮我交fee_id为3的费用" | 需先查询获取fee_id，防止重复缴费 |
| 5 | `query_notices` | 查询通知 | "最近有什么通知？" | 显示最新5条通知，紧急通知会标注 |
| 6 | `query_fee_settings` | 查询收费标准 | "水费多少钱一吨？" | 显示物业费/水费/电费/燃气费单价 |
| 7 | `query_owner_info` | 查询业主信息 | "我的房号是多少？" | 显示姓名/电话/楼栋/单元/房间号 |
| 8 | `evaluate_repair` | 评价维修 | "给维修打个5分" | 不指定工单ID时自动评价最近已完成的工单，仅已完成工单可评价 |
| 9 | `query_activities` | 查询活动 | "最近有什么活动？" | 显示活动标题/时间/地点 |
| 10 | `submit_feedback` | 提交反馈 | "我要投诉楼道灯不亮" | 提交意见反馈，状态为"待处理" |
| 11 | `create_forum_post` | 论坛发帖 | "帮我发个帖子：寻物启事" | 不指定分区时自动根据内容判断分区 |
| 12 | `search_forum_posts` | 搜索帖子 | "帮我搜一下关于停水的帖子" | 模糊搜索标题和内容 |

### 🔧 管理员专用工具（5个）

| # | 工具名 | 功能 | 示例触发语 | 说明 |
|---|--------|------|-----------|------|
| 11 | `dispatch_repair` | 派单 | "把工单5派给张师傅" | 仅"待审核"工单可派单，需提供维修员ID |
| 12 | `query_available_workers` | 查询维修员 | "有哪些维修员空闲？" | 列出所有状态正常的维修员及联系方式 |
| 13 | `query_dashboard_stats` | 系统概览 | "帮我看看系统概况" | 业主数/维修员数/报修统计/缴费率/通知数/活动数 |
| 14 | `urge_payment` | 一键催缴 | "催一下没交费的业主" | 批量标记催缴提醒，返回催缴数量 |
| 15 | `publish_notice` | 一键发通知 | "发个通知：明天停水" | 自动创建通知公告，可设普通/紧急类型 |

## 四、典型对话场景

### 场景1：业主报修
```
业主: 我家厨房水管漏水了，很急！
Agent: [调用 create_repair]
       已为您创建报修单，单号#15，状态：待审核。
       管理员将尽快派维修员处理，请耐心等待。
```

### 场景2：业主缴费
```
业主: 我欠了多少钱？
Agent: [调用 query_unpaid_fees]
       您目前有3笔未缴费用：
       - 2024-01 水费: 45.00元 (fee_id: 5)
       - 2024-01 电费: 120.50元 (fee_id: 6)
       - 2024-01 物业费: 200.00元 (fee_id: 7)
       共计365.50元。需要帮您缴纳吗？

业主: 帮我交水费
Agent: [调用 pay_fee(fee_id=5)]
       缴费成功！水费45.00元已缴纳。
```

### 场景3：管理员派单

## 五、权限控制

Agent根据JWT Token中的用户角色自动控制可用工具：

| 角色 | role值 | 可用工具数 | 可用工具 |
|------|--------|-----------|---------|
| 管理员 | 0 | 17个（全部） | 业主工具 + 管理员工具 |
| 业主 | 1 | 12个 | 业主工具（不含派单/催缴/发通知等） |
| 维修员 | 2 | 5个 | 仅查询类工具（报修查询/业主信息/通知/活动/反馈） |

> 业主调用管理员工具时，Agent不会将该工具发送给LLM，LLM自然不会调用，确保权限安全。

## 六、技术架构

```
用户消息
   │
   ▼
AgentController (/agent/chat)
   │ 提取 userId + role (from JWT)
   ▼
PropertyAgent.chat(userId, role, message, history)
   │
   ├─ 根据role过滤工具列表 (getToolListByRole)
   ├─ 根据role生成系统提示词
   │
   ▼
LLM API (OpenAI Function Calling协议)
   │
   ├─ LLM返回tool_calls → 执行工具 → 结果回传LLM → 最多5轮
   └─ LLM返回content → 最终回复
   │
   ▼
返回 { reply, tool_calls, rounds }
```

| PropertyAgentTools.java | com.wye.agent | 工具定义与执行 |

## 七、配置要求

1. **AI模型需支持Function Calling**：推荐使用 Qwen系列、GPT-4、GPT-3.5-turbo 等支持tool calling的模型
2. **AI配置需启用**：数据库 `ai_config` 表中需有 `enabled=1` 的记录
3. **Agent配置**（application.yml）：
   ```yaml
   agent:
     max-tool-rounds: 5    # 最大工具调用轮数
     max-history: 10       # 最大对话历史条数
   ```
